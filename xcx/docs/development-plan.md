# 微信小程序开发文档

## 1. 项目背景

当前仓库已经完成网站端原型，核心内容集中在以下能力：

- 首页：介绍系统价值，强调产地识别、营养预测、异常预警。
- 数据看板：查看样本总量、产地分布、糖度趋势、异常样本、近期检测记录。
- 智能分析：上传高光谱数据，进行单样本分析或对比分析，输出 AI 文本报告。
- 历史记录：按样本编号、产地筛选历史检测数据，并支持导出。
- 模型展示：展示分类/回归模型表现、Explainable AI、特征波段重要性。
- 技术介绍：说明高光谱成像、深度学习模型与系统优势。

微信小程序端不是简单复刻网站页面，而是要在移动端形成一套可以实际使用的业务闭环。

## 2. 小程序建设目标

目标是新建独立的 `xcx` 工程，使微信小程序端与现有网站端并行存在，且共用统一业务数据。

小程序端需要覆盖以下真实业务能力：

1. 登录并识别当前微信用户。
2. 查看产地样本、品质指标、历史分析记录。
3. 发起样本分析任务，查看分析结果与 AI 报告。
4. 支持单样本分析和双样本对比分析。
5. 提供移动端可读的数据总览和产地洞察。
6. 后端独立部署，数据库独立管理，不依赖当前网站页面逻辑。

## 3. 建设原则

- 网站端保留不动，不把网站代码强行改造成小程序。
- 小程序端独立设计，交互更偏移动端和业务操作。
- 后端统一提供 API，未来网站端也可以逐步接入同一套接口。
- 数据库存储正式业务数据，不再长期依赖前端 mock 数据。
- 当前数据模型先围绕橙子样本、产地、分析任务、分析结果设计，后续支持扩展。

## 4. 小程序功能范围

## 4.1 V1 必做功能

### A. 首页总览

展示：

- 样本总量
- 产地数量
- 预警样本数量
- 最近分析记录
- 重点产地表现
- 质量分布摘要

目的：

- 让用户一进小程序就能看到当前业务数据概况。

### B. 样本列表

支持：

- 按样本编号搜索
- 按产地筛选
- 按状态筛选
- 列表分页
- 进入样本详情

目的：

- 作为小程序的主数据入口。

### C. 样本详情

展示：

- 样本编号
- 所属产地
- 采样时间
- 糖度 SSC
- 酸度 TA
- 糖酸比
- 维生素 C
- 历史分析记录

支持：

- 发起一次新的分析任务
- 查看该样本近期分析结论

### D. 智能分析

V1 在小程序端的定义：

- 支持单样本分析
- 支持双样本对比分析
- 支持查看 AI 文本报告
- 支持查看预测产地、置信度、核心指标

说明：

- 真正的高光谱文件上传与模型推理可先预留接口。
- V1 可以先用“根据样本触发后端分析任务”的方式打通闭环。

### E. 历史记录

支持：

- 查看所有分析记录
- 按时间倒序展示
- 查看记录详情
- 后续支持报告导出

### F. 洞察页

展示：

- 各产地综合表现
- 甜度/酸度/维 C 统计
- 稳定性评分
- 推荐产地结论

### G. 用户中心

支持：

- 微信登录
- 查看当前登录身份
- 后续扩展“我的分析记录”“我的导出报告”

## 4.2 V2 可扩展功能

- 真正上传高光谱文件
- 调用模型服务做实时推理
- PDF 报告导出
- 批量样本任务
- 设备接入与扫码采样
- 角色权限区分
- 果园/批次级追溯

## 5. 页面结构建议

建议采用 4 个 TabBar 页面：

1. 首页 `pages/home/home`
2. 样本 `pages/samples/samples`
3. 洞察 `pages/insights/insights`
4. 我的 `pages/profile/profile`

非 TabBar 页面：

- 样本详情 `pages/sample-detail/sample-detail`
- 分析结果详情 `pages/analysis-detail/analysis-detail`
- 对比分析页 `pages/compare/compare`
- 报告预览页 `pages/report/report`

## 6. 后端模块划分

后端建议独立放在 `xcx/server`，职责如下：

### 6.1 认证模块

- 微信登录
- 获取 `openid`
- 用户信息落库
- 生成会话信息
- 校验登录态与用户归属

当前进展补充：

- 已完成签名会话 token 与基础接口鉴权

### 6.2 样本模块

- 样本列表
- 样本详情
- 条件筛选
- 分页查询

### 6.3 分析模块

- 发起分析任务
- 查询任务状态
- 生成分析结果
- 返回 AI 文本报告

当前进展补充：

- `POST /api/analysis/tasks` 已切为异步任务入口
- 服务启动时会自动恢复 `PENDING / RUNNING` 任务

### 6.4 洞察模块

- 统计产地表现
- 汇总甜度、酸度、维 C
- 产地排行
- 预警统计

### 6.5 报告模块

- 报告生成
- 报告下载
- 报告归档

### 6.6 基础数据模块

- 产地管理
- 果园管理
- 批次管理
- 模型版本管理

## 7. 推荐技术方案

## 7.1 小程序端

- 方案：原生微信小程序
- 原因：最直接，依赖最少，适合快速落地和后续接微信能力

## 7.2 后端

- 运行时：Node.js
- 框架：Express 或 NestJS
- 当前建议：先用 Express 快速搭建，再视复杂度升级

## 7.3 数据库

- 生产推荐：MySQL 8.0
- 开发环境可先用 SQLite 或本地 MySQL
- ORM 推荐：Prisma
- 当前后端已补齐 SQLite / MySQL 双目标 Prisma 命令，便于开发与生产切换

## 7.4 存储

- 数据库：业务结构化数据
- 对象存储：高光谱原始文件、PDF 报告

当前进展补充：

- 已补齐本地文件存储抽象层
- 文件下载已切为带鉴权的后端接口

## 7.5 模型服务

后续预留：

- Python 推理服务
- HTTP/RPC 调用
- 输入为光谱文件或特征值
- 输出为产地分类、营养指标、解释文本

## 8. API 规划

建议的首批接口如下：

- `POST /api/auth/wechat/login`
- `GET /api/home`
- `GET /api/insights`
- `GET /api/origins`
- `GET /api/samples`
- `GET /api/samples/:sampleCode`
- `POST /api/analysis/tasks`
- `GET /api/analysis/tasks/:taskId`
- `GET /api/analysis/results/:resultId`
- `GET /api/reports/:reportId`

## 9. 数据流说明

### 9.1 查看样本

1. 小程序发起样本列表请求
2. 后端从数据库读取样本、产地、统计信息
3. 返回轻量化数据给小程序

### 9.2 发起分析

1. 用户从样本详情点击“开始分析”
2. 后端创建分析任务
3. 调用分析逻辑或模型服务
4. 写入分析结果表
5. 小程序轮询或刷新查看结果

### 9.3 登录

1. 小程序调用 `wx.login`
2. 拿到 `code`
3. 提交给后端
4. 后端换取 `openid`
5. 用户信息落库并返回会话信息

## 10. 开发顺序建议

### 第一阶段

- 明确数据库表
- 搭后端基础工程
- 完成认证、样本、洞察接口
- 建立种子数据

### 第二阶段

- 完成小程序页面骨架
- 接通列表、详情、登录、分析接口
- 完成移动端交互和视觉设计

### 第三阶段

- 接真实文件上传
- 接模型推理
- 接报告导出
- 完成测试和部署

当前进展补充：

- 文件上传已接入
- 报告 PDF 导出已接入
- Node 后端已支持切换到 Python HTTP 模型服务进行联调

## 11. 当前结论

基于现有网站内容，这个小程序项目的核心不是“再做一个宣传页”，而是做一套移动端样本分析系统。

它至少要覆盖：

- 数据总览
- 样本管理
- 分析任务
- 分析结果
- 洞察统计
- 微信用户接入

后端和数据库必须作为正式项目的一部分一起建设。

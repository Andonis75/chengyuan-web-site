# 橙源智鉴 - 小程序前端开发计划与设计规范

## 1. 设计理念与视觉风格

为了区别于网页端，并充分发挥移动端的交互优势，小程序端将采用全新的视觉设计语言。

### 1.1 色彩规范 (Color Palette)
坚决弃用蓝紫渐变，采用契合“橙子”主题的温暖、自然、科技感并存的色调。

*   **主色调 (Primary)**: ` #FF8C00` (活力橙) - 用于主要按钮、高亮文字、核心图标。
*   **辅助色 (Secondary)**: `#FFA500` (明亮橙) - 用于渐变过渡、次要强调。
*   **背景色 (Background)**: `#F7F8FA` (极浅灰) - 作为全局背景，使卡片内容更突出。
*   **卡片背景 (Card)**: `#FFFFFF` (纯白) - 用于承载具体内容的卡片。
*   **文本色 (Text)**:
    *   主要文字: `#333333` (深灰)
    *   次要文字: `#666666` (中灰)
    *   辅助文字: `#999999` (浅灰)
*   **状态色 (Status)**:
    *   成功/正常: `#07C160` (微信绿)
    *   警告/异常: `#EE0A24` (警示红)

### 1.2 视觉风格 (Visual Style)
*   **卡片化设计 (Card-based UI)**: 采用大圆角（如 `border-radius: 16rpx`）的卡片来组织信息，增加页面的呼吸感和层次感。
*   **微阴影 (Soft Shadows)**: 为卡片添加轻微的阴影（如 `box-shadow: 0 4rpx 16rpx rgba(0,0,0,0.04)`），提升立体感。
*   **留白 (Whitespace)**: 增加元素间的间距，避免信息拥挤，提升阅读体验。
*   **无边框 (Borderless)**: 尽量减少生硬的边框线，通过背景色差和阴影来区分层级。

## 2. 技术选型与架构

*   **框架**: 微信小程序原生框架 (WXML + WXSS + JS)。
*   **UI 组件库**: 核心页面采用自定义编写以保证视觉独特性，部分标准交互（如弹窗、Toast、Loading）可引入轻量级组件或自行封装。
*   **网络请求**: 封装 `wx.request`，统一处理 BaseURL、Token 注入、错误拦截。
*   **状态管理**: 简单的全局状态（如用户信息、Token）存放在 `app.js` 的 `globalData` 中，并结合 `wx.setStorageSync` 进行持久化。

## 4. 开发阶段划分

### 阶段一：基础设施与全局配置 (当前进行中)
1.  初始化 `app.json`，配置 4 个 TabBar 页面。
2.  编写 `app.wxss`，定义全局 CSS 变量（颜色、字体大小、圆角等）。
3.  封装 `utils/request.js`，打通与本地 Node.js 后端的通信。
4.  实现微信登录逻辑，获取并存储 `sessionToken`。

### 阶段二：核心 TabBar 页面开发
1.  **首页 (`home`)**: 顶部欢迎语 + 核心数据概览卡片 + 快捷入口 + 最近动态列表。
2.  **样本 (`samples`)**: 顶部搜索/筛选栏 + 样本列表（卡片式布局，展示关键指标）。
3.  **洞察 (`insights`)**: 产地排行 + 质量分布（使用简洁的进度条或环形图展示）。
4.  **我的 (`profile`)**: 用户信息展示 + 历史任务入口 + 设置。

### 阶段三：业务详情与交互链路
1.  **样本详情 (`sample-detail`)**: 详细指标展示 + 历史分析记录 + 发起分析按钮。
2.  **发起分析 (`analysis`)**: 选择对比样本 + 确认分析配置。
3.  **分析结果 (`result`)**: AI 报告展示 + 核心指标对比。

### 阶段四：细节打磨与动效
1.  添加页面切换动画、按钮点击反馈。
2.  完善 Loading 状态和骨架屏 (Skeleton)。
3.  处理各种异常状态（无网络、接口报错、数据为空）。
